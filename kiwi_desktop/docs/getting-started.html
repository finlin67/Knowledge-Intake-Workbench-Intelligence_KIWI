Quick start
Your first five minutes
Follow these steps in order. Most users have their first export done in under 10 minutes.
STEP 01
Name your project & set folders
Step 1 panel
Enter a project name (anything works — "MyDocs", "CareerFiles", "Q3Archive"). Set your Source Folder to the folder containing your documents, and your Output Folder to wherever you want the organized export to land.
Project Name field Source Folder → Browse Output Folder → Browse
STEP 02
Scan your folder
Step 3 panel
Click Create + Load + Scan — this single button creates your project, loads it, and scans all documents in your Source Folder. KIWI reads every PDF, Word doc, PowerPoint, and markdown file it finds. This takes a few seconds to a few minutes depending on how many files you have.
+ Create + Load + Scan
STEP 03
Review the Triage tab
Expert mode
Switch to Expert Mode using the toggle in the status bar, then open the Triage tab. The five stat cards at the top tell you how many files need attention. Files marked "High priority" or "Rule gaps" are the ones worth reviewing — KIWI wasn't sure where to put them. You can assign them manually or skip them for now.
Toggle Expert Mode Triage tab Filter by Priority
STEP 04
Run the export
Step 3 panel
Click Run Queue to export your classified documents to the Output Folder. Watch the Run Monitor tab for progress. When complete, your Output Folder will contain organized workspace subfolders ready to import into AnythingLLM or Open WebUI.
Run Queue Run Monitor tab Run Both Profiles
STEP 05
Point your AI tool at the output
Outside KIWI
In AnythingLLM or Open WebUI, create a workspace and upload or point it at your Output Folder. Your documents are now organized, chunked, and ready to query. Ask your AI assistant anything about your professional history, projects, or expertise.
AnythingLLM workspace Open WebUI collection
Pro tip
Use Run Both Profiles in Step 3 to export in both AnythingLLM and Open WebUI formats simultaneously. Try both and see which your AI tool retrieves from more accurately.

AI setup
Choosing an AI provider
KIWI works without any AI at all. Add a provider to classify files that don't match your rules.
Free · Local
Ollama
Runs entirely on your computer. No API key, no cost, no data leaving your machine. Best for privacy-conscious users.
  • Download Ollama from ollama.ai
  • Run: ollama pull llama3.2:3b
  • In KIWI Settings → set AI Provider to ollama
  • Click Test Ollama Connection
API key required
Claude (Anthropic)
High accuracy classification. claude-haiku is fast and cost-effective for bulk document processing.
  • Get API key at console.anthropic.com
  • In KIWI Settings → AI Provider → claude
  • Paste your API key
  • Select claude-haiku for speed/cost
API key required
OpenAI
Widely recognized models. gpt-4o-mini is recommended for classification — accurate and affordable.
  • Get API key at platform.openai.com
  • In KIWI Settings → AI Provider → openai
  • Paste your API key
  • Select gpt-4o-mini
You don't need AI to get started
KIWI's rule-based classifier handles the majority of files for most users. Go to Settings → My Categories and add your company names, project keywords, and topic phrases. Files matching those rules are classified instantly — no API calls, no cost, no latency.
Large archives
Working with 1,000+ files
Use the Archive Prep Tool to split large folders into manageable batches before scanning.
Recommended batch sizes
Under 1,000 files → no batching needed.
1,000–5,000 files → 500 files per batch.
5,000–20,000 files → 300–500 per batch.
20,000+ files → 200–300 per batch.
# Run the Archive Prep Tool standalone
python archive_prep_tool_gui.py

# Or launch it from inside KIWI using the Prep Tool button in Step 2
The Prep Tool removes empty files, splits your archive into numbered batch folders, and generates a CSV manifest. Point KIWI's Source Folder at one batch at a time. When KIWI finishes a batch, move to the next.
FAQ
Common questions
What does "unassigned" mean in the Triage tab?
KIWI couldn't confidently route the file to a workspace using your current rules. You can assign it manually, add a rule to handle similar files automatically next time, or mark it to skip. "Safe to skip" files are typically empty, tiny, or duplicate — KIWI flags them so you don't have to review each one.
Do I need an AI provider to use KIWI?
No. KIWI works entirely on rules you define in Settings → My Categories. AI (Ollama, Claude, or OpenAI) is optional — it's only called for files that don't match any of your rules. Most users find that well-configured rules handle 80–90% of their files without any AI at all.
What's the difference between AnythingLLM and Open WebUI export?
AnythingLLM gets a flat folder of markdown files — all documents in one directory. Open WebUI gets a folder hierarchy that mirrors your workspace structure. Use Run Both Profiles to generate both simultaneously, then try each with your AI tool to see which retrieves more accurately for your use case.
What is Simple vs Expert Mode?
Simple Mode shows the three tabs you need for most runs: Queue, Run Monitor, and Settings. Expert Mode reveals the full tab set — Inventory, Review, Triage, Queue, Run Monitor, Exports, and Settings. Start in Simple Mode and switch to Expert once you're comfortable with the basic workflow.
What happens if I click Stop mid-run?
KIWI saves all work completed up to that point. You can Resume later to continue from where it stopped, or start a new Run — nothing is lost. Pause temporarily halts the run without closing it; Stop ends the active loop.
How do I teach KIWI about my specific documents?
Go to Settings → My Categories. Add your company names under Companies (e.g. "boeing", "lockheed"), your project names under Projects, and specific document keywords under Keywords & Rules. KIWI uses these to classify files deterministically — faster and more accurate than AI for your known content.
Can I use this if I'm not a developer?
Yes. KIWI requires Python 3.11 to install, which is a one-time download. After that, everything is a point-and-click desktop application. A packaged Windows .exe installer (no Python required) is on the roadmap for future releases.
Glossary
Plain English definitions
Terms you'll encounter in KIWI and what they actually mean.
Workspace
A top-level folder category. Files are sorted into workspaces (e.g. career_portfolio, ai_projects, archive). You define which workspaces exist in Settings.
Force Rule
A keyword that always routes a file to a specific workspace — no AI needed. Highest priority classification rule. If a filename or content contains "resume", it goes to career_portfolio.
Triage
Files KIWI couldn't confidently classify using your rules. Reviewing these and adding new rules improves every future run.
Batch
A subfolder of files processed in one KIWI run. Created by the Archive Prep Tool when splitting large archives.
Export Profile
AnythingLLM (flat file output) or Open WebUI (folder hierarchy). Controls how your exported knowledge base is structured.
Confidence threshold
How certain KIWI must be before auto-assigning a file. Set in Settings. Lower = more auto-assignments, less manual review required.
Rule gap
A file that didn't match any rule in your config and had to fall back to AI or manual review. Adding a rule for it prevents future gaps.
Subfolder
A second-level folder inside a workspace. For example, career_portfolio/resumes or archive/abm. Defined in your Force Rules.
Preflight summary
The report shown in Run Monitor before you click Run. Shows total files in scope, pending count, AI mode, and export target.
