# KIWI Documentation Map

## Purpose of each doc

- README.md: GitHub-facing overview of the combined product and repo layout.
- QUICK_START_WINDOWS.md: fastest beginner path to run KIWI on Windows.
- USER_GUIDE.md: full operator guide for normal day-to-day runs.
- BEGINNER_WALKTHROUGHS.md: scenario-based click-by-click walkthroughs.
- RELEASE_CHECKLIST.md: V1 release validation checklist.
- RELEASE_WINDOWS_GUIDE.md: maintainer publishing workflow.

## Subproject docs

- KIWI_Web: web UI code and in-app help content.
- kiwi_desktop: local backend/tooling docs and architecture notes.

## Which folder do I use?

- Non-technical users: use the parent KIWI folder and start_kiwi.bat.
- Operators: work mainly in the browser UI (Home/Setup, Settings, Triage, Inventory).
- Developers: edit KIWI_Web and kiwi_desktop separately.
